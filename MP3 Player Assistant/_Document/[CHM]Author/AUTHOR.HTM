MP3 Player Assistant 作者自述

2005 年 5 月 21 日
陈明龙

　

陈明龙，出生于中国著名的火炉之城――武汉，今年 18 岁，就读于武汉市第四十高级中学高三(1)班。喜欢写程序，但不喜欢玩游戏（很奇怪吧），喜欢网络，不喜欢聊天，喜欢 P2P，不喜欢 HTTP。　

　

　

懵懵懂懂的小学生活

说起玩电脑，我可以说是我们班上玩电脑最早了。从小学二年级开始，我们家中就买了一台 286 的电脑。但是当时我玩得还很少，当时就是玩一下 WPS，学一下打字，很快我的拼音在班上是最好的了，这还要归功于电脑。后来到了四年级，我们家的电脑首次装上了当时的奢侈品――硬盘，是一个 256 MB 的硬盘（比我家现在地内存还小）于是在首次见到 C:\> 的命令符下，学习了 CONFIG.SYS 和 AUTOEXEC.BAT 的配置，学会了自己改造系统。父亲当时亲自教我一些 DOS 命令，我也觉得很过瘾，应为当时很小，特别是 DIR 又简单，又可以一下子将整个屏幕填满，当时这使我很有成就感。还记得当时最难道命令莫过于 EDLIN，是一个编写文档的命令，如果你现在使用 ENLIN，你一定会感慨原来在 DOS 下编写一个文本文件是一件多么困难道事情（当时还没有 EDIT 命令）。我当时学习的时候为了总结所学习的知识，还用电脑写了两本小节，当时所用的软件是 UCDOS + WPS，现在看来当时的软件还真落后。

后来，小学终于结束了。我还记得在小学毕业的时候家中买了一台新电脑，配置在当时还是很高的，赛扬 333 的 CPU，64 MB 内存，6.4 GB 硬盘，还有 32x 的光驱，于是我终于结束了 Windows 3.2 和 UCDOS，来到了很成熟的 Windows 98 系统。在 Windows 98 下我继续完成了我小学的梦想――写一本书，我使用当时很流行的 Word 97 继续编写完的小学的小节，我终于发现原来还有这么好用的软件，比起 WPS 要好用多了，第一次感觉到“所见机所得”的快感，但是编写的结构图的时候遇到了很多麻烦――只要移动图片的，所有的内容的位置都会改变，搞得不好就把你辛辛苦苦编写的文档变得面目全非。后来我终于发现了一个办法，就是在文档中插入一个 Word 图片对象，然后编辑这个图片对象，而不是在文档中直接插入图片或线条，现在的 Word 2003 简化了这个动作，只要你插入一个线条它就自动创建一个 Word 图片对象。终于在初中开学的时候，我将《小学语文小节》和《小学数学小节》打印出来了，语文小节有75页，数学小节有38页，一共只打印了三份，将其中的两份赠送给我的启蒙导师，也是我认为对我影响最大的老师――张敏老师和叶东平老师，他们是我小学2－4年级的语文老师和数学老师。直到现在我还保存着这两本书。

在小学毕业的三个月中，别人都铆足了劲出去玩，我却在电脑旁待了三个月，没有导师辅导、没有参考书，只有帮助文件的我不断地尝试，除了 Word 中的 Visual Basic for Application 没有学会，其他的东西都学会了，并且初步了解了 Excel 和 PowerPoint。初中即将开学，我还在家里玩电脑，而有些人都上什么衔接班，我没有去。我在家玩 Access 数据库，当时的窗体、模块我是一窍不通，所有窗体上的功能都是使用一个简单的功能――宏来代替的。Access 中的宏是一个很好用的功能，他把编写应用程序的复杂方案简化了，把代码变成了图形结构。但是有一个很重要的弊端，它没有 Visual Basic 的程序的扩展性，也没有 Visual Basic 的易读性。我认为 Access 是 Office 软件套装中最难懂得一个软件，比起 Word、Excel 和 PowerPoint 它有太多东西难懂，比如查询中的 SQL 语句，窗体中的代码编辑器等。

比较可笑的是，我当时连软件分类都分不清楚，家中有好多软件都没有用过，比如ACDSee和 WinZIP等。它们相对于 Office 家族，是在是太简单了，我回到家后居然连怎么播放 MP3 都不知道，还是父亲教我的。它买了一张钢琴曲的 MP3 的碟子，然会将文件一拖就复制到硬盘上，我当时很惊奇，以前需要使用 Copy 命令，现在“一拖”就解决问题了。我父亲因为一般都不会使用压缩，所有当我问起他 WinZIP 的时候，他不知道，于是我自己学习了压缩技术。这可是一款我第一次接触到新软件区域的软件。
　

　

程序设计幻想的初中前期

在众多编程软件中我选择了 Visual Basic 是一件偶然的事。为了让我们家都学习电脑，父亲花了一笔钱买来全套《开天辟地》软件，我在家的时候就自己看，我发现头三盘的东西我都已经了解了，比如如何开机关机，如何使用记事本、造字程序等。直到了最后的一张光盘，它是讲述程序设计初步和多媒体制作的，我使用了这么多的程序，也一直希望自己制造一个程序，于是这张光盘就强烈地吸引了我，我开始按照其中的一步步试着制作程序，画界面，写代码，但是第一次当我满怀喜欢的运行的时候，VB 却提示我“变量未定义”这一错误的提示，这对于一个初中一年级的学生来说这与没有提示是一回事，于是我开始的最初很弱智地Debug――就是对照《开天辟地》中的步骤一步步的来，可是在完全一样的程序下，我却仍然出现了“变量未定义”这一错误的提示，无奈之下，我只好单击“帮助”，这也都是一些我看不懂的东西，在花了几天的时间后，我终于找到了错误的原因――在 Visual Basic 中，在调用任何一个变量之前，必须对它定义，确定变量的数据类型和分配的空间。而 Visual Basic 的选项中也有一项“要求变量声明”，如果没有选择它的时候并且模块的声明部分没有“Optional Explicit”语句，就会将语句中的没有声明变量的字符自动声明。但是这个选项默认是打勾的，所以我在语句中调用了一些变量却没有声明，这样就引发了“变量未定义”的错误。

在整个初一，我几乎都只是在学习 Visual Basic，几乎没有写出一个像样的程序，也没有很多时间接触电脑了。因为我在初一的时候，发现初中的科目越来越难了，还有抽时间看看奥林匹克的相关内容，当时我在班上的成绩是很优秀的，我不能因为电脑尔忽视的我的成绩，毕竟我是学生，而且是在这个号称“惟楚有才”的武汉。

说起来，我初二还真是胆大，没有学习结构、没有学习类的封装、没有了解集合，没有看过FileSystemObject 对象。我居然大胆地向开发一个 ZIP 的压缩软件，因为我当时很不喜欢 WinZIP 的界面，因为我用资源管理器用惯了，希望有一个“树”来方便查看目录，并且 ZIP 格式支持路径的管理，但偏偏 WinZIP 不支持树的查看。没有分析 ZIP的文件格式，不知道压缩的核心算法，我便使用了一个 ActiveX 控件来解读 ZIP 文件，可笑的是，当时的数据封装居然是将数据放在一个隐藏的带有 ListView 控件的窗体中，一开始将所有数据放在中，然后在调用 Item 的 Text 属性和 SubItems 属性来读取数据。如果是现在的我，我一定会按照大多数的思路，首先创建两个类 ZipFile 类和 ZipFiles 类，在 ZipFile 类中添加要存储的属性，然后将 ZipFiles 改成一个集合类，将 ZipFile 类作为这个集合的元素。在这样“低级”的算法中我居然写出了分析 ZIP 文件夹的功能，并且总后实现了浏览的功能。可笑的是，我当时使用的 Xceed Zip 控件居然是一个使用版本，没法，试用期过后，我就没有在编写这个程序了，代码，也就删除了。

后来，VCD 在我们城市兴起了，家里人希望搞一个 VCD 出租的店子，也希望将家中的电脑利用出来，我自告奋勇地开发一个软件管理 VCD 出租的软件。也许是 Access 数据库的成熟，我使用了 VB+DAO 的数据访问方式开发。这一次是头一次规划了软件的结构、功能，并且自己设计了良好的界面，第一次使用了 ctGrid 控件。这个程序不但具有了出租、归还、损坏、进货和废弃等基本功能，还有会员制、自动排序热门影碟等功能，还具有完整的用户管理功能，并且限制了具体的权限。但是在编写到一半，我发现核心仍然不完美，因为数据封装没有使用到最高级的类模块+集合模式，依然是数组+用户自定义类型模式，所以在查询的时候，它的遍历模式没有关键字搜索快。而且在编写最后一项时，家人又取消了原来的计划，所有这个程序也是一个半成品而已。

第一个真真正正的成品

虽然以前的作品已经都失败了（PS：之前的程序不仅仅就只有以上两款，还有 RTF 编辑器和打字先锋，因为年代久远，源代码也没有了，连基本的思路都没有了，所以在这里就不介绍了），但是它们给我的程序设计的思路却是很丰富的，正所谓失败乃成功之母，在那以后，我已经积累了不少 Visual Basic 的核心内容，从类模块到 Win32API，从窗体到 DHTML，无不设计，并且我已经知晓在开发一个大程序之前要做些什么，必须做些什么。

HTML 帮助文件在2000 年兴起，并且获得了良好的评价，但是 Microsoft 自带的 HTML Help Workshop却不是那么好用，因为它的界面是英文版的，而且即使是网上有中文的汉化版本，可是谁又能汉化它的帮助文件呢？而且其中的功能复杂，即使是网上有一些电子书的制作软件，都没有 HHW 的功能多，大多都是吹嘘自己的功能，连最基本的上下文弹出帮助都不能制作，最基本的多重链接（A-Link）都无法胜任，还要收费。于是我便酝酿了一个计划，开发一个制作 HTML 帮助文件大程序套件，就像 Office 一样，分别可是实现弹出帮助和多重链接。和其它的程序不一样，我首先开发的是其核心部分，开发类和集合，这是数据封装的基本内容。最后通过几个月的开发，其中的一个套件问世了，Easy HTML Help Popup 就是其中的一个开发套件。发布到网上，可是没有多少人问津，可能是因为应用的局限性太小了吧。所以虽然这个程序开发完成了，剩下的套件就没有下文了。

这次之后，我终于了解到要开发一个好的应用程序是一件多么困难道事情，而且，如果使用面不广的东西坚决不要开发，这就是浪费你的精力和时间。

　

　

从应用程序到 ActiveX

初四来临了，我也几乎没有什么时间玩电脑了，也没有大块大块的时间来写程序了，面临中考――这一人生大关，压力和学习逐渐占领了整个我，除了周末玩两个小时的《帝国时代2》再也没有其它剩余的时间了。最终皇天不负有心人，我以绝对优势的分数进入了武汉市第十四中学，一个省重点高中的快班，并且获得了奖学金。这样我也终于有可以开始写程序了，又可以开始做我喜欢做的事情了。

制作 EHHP 的时候，我就开始使用了 ActiveX，我越来越发现 ActiveX 是一个很好的东西，很多人都可以来分享自己的成果到自己的应用程序中，我年轻也有这样的冲动――为人命服务。于是我开始了第一个 ActiveX DLL，它是封装 INI 文件的 ActiveX DLL，可以轻松地像 DAO 一样调用 INI 中存储的信息，而且完全的数据封装，数据缓存，可以大大加快对 INI 文件的访问，这也是我现在一直使用的 ActiveX 之一。

随着 Windows XP 的盛行，它的界面深深地吸引了我，特比是资源管理器中的那个漂亮的滑动菜单，这个控件如果编写出来，那该是一个多么好的产品呀！于是我在网上搜索，居然没有一个控件可是实现该功能，vbAccelerator.com 提供了一个很好的方案可是只能在 WinXP 下运行，因为它是通过 API 的方式直接调用 comctl32.dll 而实现的。于是我便制作了一款这个 ActiveX OCX，它几乎扩展了很多功能，而且也受到网友的好评。

天时地利人和

重点高中就是不一样，我以前在初中，由于学校不怎么样，大多数的同学和老师都只认为我只会打字，只认为除了打字我的电脑不怎么样，他们评定我的标准就是因为我玩游戏很菜。可是到了高一下学期，当我们上研究性学习时，老师要求我们选定一个课题，我选定了“网页制作”，到学期末的时候，我终于制作出一份像样的作品，我原来在编写帮助文件的时候曾经学习过 Dreamweaver，所以有写网页的基础，而且原来我也曾经制作过一两个网站，我又是一个特别注重程序界面的人，所以对 Photoshop 和 CorelDraw 也不陌生。这个网站在高一学期末上交了，我的研究性学习课题得了满学分。而且我的电脑技术也得到了学校的认可。在这天时地利的情况下，我终于有了一点气候。

自己设计的网站――师生驿站


突然，在高二的时候，老师通知我有一个电脑作品大赛，全称是《第六届全国中小学电脑制作比赛活动选拔赛》，希望我参加，于是我报了名，因为时间有半年所以比较充足。于是我详细制定了作品计划，这一次我要用十二分力气来完成这件事情，通过半年的时间，我终于写出了我有生以来最引以自豪的软件《游戏管家》，由于功能很多内容复杂，我就直接以引用说明文件了：
　

（以下引用自《游戏管家》说明文件）
Game Manger 1.0 是一个游戏管理、控制软件。它可让玩家玩的游戏尽在掌握之中，强大的分析功能可以使您完全掌握玩家的游戏动态，完备的保护监视的机制，可以使得您完全不必担心玩家会跳出游戏管家的控制。同时她还提供了强大的时间计划设置，这些设置可以控制在不动时间不动时段，限制玩家所玩的游戏。

Game Manager 有五个组件：
★ Game Manager Administrator 是游戏管家的主体程序，她是 Game Manager 的核心，她控制者添加游戏、删除游戏、卸载游戏和强大的图表组件，可以让你以任何方式查看图表、分析数据。同时可以对游戏管家进行全局控制，例如节假日设定，玩家添加，时间计划的设定，备份数据库等。
★ Game Manager Starter Center 是游戏管家的启动游戏的主体程序，通过它可以和 Game Manager Watcher 互相通讯，监视进程后开始游戏。同时它可以在游戏进行的同时监视游戏，最后将游戏玩的事件添加到数据库中。她的界面上可以以分类试图、筛选试图来打开游戏，并且如果游戏超过了时间计划定义的时间，她就不会让游戏运行。如果没有添加游戏，它是不会启动的。
★ Game Manager Watcher 是游戏管家的监视中心，它可以随时监视系统进程的运行如果有游戏进程启动，会马上终止它，并且启动 Game Manager Starter Center，使用它来启动游戏，防止玩家使用资源管理器或 Console 模式运行游戏。如果运行它之后再运行它，会终止 Game Manager Watcher（需要管理员密码）。
★ Game Manager Prompting Application 是游戏管家的提示中心，它可以在游戏的运行的时候提示一些信息，比如：当游戏时间用完的时候它会按照设置的属性进行提示，可以播放声音、朗读英文（需要 Microsoft Text-to-Speech）、前台提示条等提示方式，同时它与 Game Manager Stater Center 是不同的进程，所以关闭它不会对整个管理系统进行全局的影响。
★ Game Manager Protector Application 是进行保护的进程，每当 Game Manager Stater Center 或 Game Manager Watcher 启动的时候，它会自动启动，防止玩家使用 Windows 任务管理器或其它进程工具非法终止监视器或启动器。

　

（以下引用自《第四届武昌区中小学电脑制作活动暨第六届全国中小学电脑制作比赛活动选拔赛推荐作品登记表》）
创作思想（解释作品创作的背景、目的和意义）
电脑已经开始走进千家万户，几乎每一台电脑上都是很多游戏，但并不是每位同学都能控制自己玩游戏的时间，对此很多家长无奈，在此本作品为监控、管理、分析游戏提供了方便，她实用、易用、方便，她使得家长不再为子女沉迷与游戏而担忧。她最后便使家长摆脱无法控制子女完游戏过度的烦恼，让游戏与我们一起健康成长。

创作过程（解释运用了哪些技术或技巧完成主题创作，哪些是得意之处）
首先本软件使用的ActiveX技术，使得每个部件都能和谐相融；使用了MS Jet和DAO 3.6控制主数据库；调用了大量的Win32 API，使用它们控制Windows进程；使用了Win32 Crypt API，对用户密码加密；使用了SSubTimer类，用于控制底层计时器；使用了双进程保护，防止用户用任务管理器终止监视器；支持MSTTS朗读英文提示，支持游戏中前端提示。
完全易懂，漂亮的界面，管理方便，图表清晰，筛选容易、只管直接写入注册表，开机自动运行（可设置）；多玩家设定；自动监视进程；双进保护；独立进程提示；强大的时间及时间段设置等。

原创部分（列明哪些部分属自己原创）
控制输入输出，进程的控制和终止，数据封装，所有的界面规划，三个图表分析，三维透视饼图，时间使用直观图，玩家条形分析图均为本人自己规划、设计。进程管理器中监视过程，计时器设定，查询ProcessID等均为自己规划，保护进程也是自己原创。

由于这个程序是参加比赛的作品，所以在比赛结果没有出来之前，我没有发布它。这个作品也许就是在天时地利人和的条件下诞生的吧。这个软件我自己也在使用，记录着我玩游戏的情况，也随时提醒我――不能沉迷于游戏。

数码时代的来临

数字生活越来越近了，最近 MP3 播放器的价格也越来越低，所以我也买的一台还算划算的 MP3。但是却没有一个软件复制歌曲到 MP3，虽然资源管理器可以复制，但是我们需要有一定的顺序，所以我就计划制作了这个 MP3 播放器的软件，因为我的格言就是“只有想不到的，没有做不到的”。
　

　

关于竞赛

信息技术奥林匹克竞赛，这可是我的痛楚。因为我荒废了初中三年，在那个初中中，没有人提过信息技术奥林匹克竞赛，我就这样荒废了三年。

到了高中，信息技术老师也要求我参加信息技术奥林匹克竞赛，第一次在高一，全校就只有两个人参加，而我报的语言是 BASIC 语言，对于我来说，Visual Basic 都已经差不多了，何况是 BASIC 语言，但是我错了，初赛容易地就通过，因为应用和理论占多数，但是复赛却都是编程序。我一看题，很简单，可是面对的是 DOS 下的 Quick BASIC，我根本就无法使用，什么对象，什么集合，什么封装，什么窗体，什么元素，都没有，就在那场考试中，我一个题目都没有答出来，真是讽刺，面对 Visual Basic 游刃有余的我却奈何不了 BASIC，更快、更优秀的东西都已经出来了，何必还要抱着老掉牙的东西不放呢？看看其它的一个个的参加竞赛的人，在开始之前个个都抱着一本本参考书，看着、读着，这是什么呀！这不是数理化，这不是史地政，这是程序设计，不是看看书就能学得好的，是要经验的积累，是要仔细地揣摩的。但是事实上，他们却获得了很好的名次，我却落榜了。

讽刺呀！学习程序设计居然可以这样学，我敢断言他们没有一个人可以写出一个真正使用的程序来。因为他们都是在钻研算法，而根本没有想要自己去做些什么，学了快速排序何必去学冒泡排序，学了多叉树何必去学二叉树。更何况，现在的这些基本算法都已经封装了，何必再去学习，写程序并不是为了学习算法，而是如何将算法组合成有用的实体。现在的信息技术奥林匹克已经无意义了。也许有人会说，没有基础拿来的发挥，可是这是程序设计，不是数理化，那为什么每本程序设计的书一开始读去介绍最简单的而表示最基本的东西。例如 Delphi 语言，一开始都是教你怎么画界面，设置属性，编写简单的代码，这些都不是基本的东西，而是最简单的东西，最基本的是每一个单元有那些语句，那些保留字，特别是 Delphi 它的美一个窗体都是一个对象，它有很多继承属性、事件，若你要从基础学习，你永远学不会 Delphi。放者现成的不用，自己去钻研已经过时的东西，累了也什么也没学会，只有两个字形容――活该！

可是这就是事实，就是这样的状况，无法改变。所以在第二年的信息技术奥林匹克竞赛的时候我报了 Pascal 语言，因为这是我比较生疏的。我花了一天的时间来熟悉 Pascal 语言、保留关键字和基本函数。令我惊讶的是，第二天的复赛，我就答了一大半的题，对于这个语言我花了很少时间认识，但是却很快地写出东西，只有一个解释，真正学会程序的人语言是相通的，只要你的思路过硬，只要你有经验，语言不是障碍。这才是真正学会程序设计的人。

　

电脑游戏

有多少人买一天电脑是为了玩游戏，虽然玩游戏也是电脑功能之一，但我始终不认为应该将玩游戏作为电脑的惟一目的。

记得在买电脑的时候，装了一个《极品飞车III》，那是我第一个玩的游戏，我觉得这个游戏没有什么，就是在公路上跑。我真正喜欢上的一个游戏那应该就是《帝国时代II》了，那是我玩的第一即使战略游戏，《星际争霸》画面太暗了，不喜欢，《红色警戒II》节奏太快，太简单，我就是喜欢节奏教慢的、养兵千日大力发展经济的游戏，所有我便选择了《帝国时代2》。特别是在一群人大战的时候，自己就像司令长一样地掌握着战局，我特别高兴。

角色扮演，不是我特别喜欢的一种，特别是回合制的，不真实。你打一拳，我打一拳，没血了喝药，没气了也喝药。没什么意思。道是剧情，如果不错，我也会玩玩，就像《仙剑奇侠传》，剧情很好，但是是悲剧，总觉得使人很不爽。我想，游戏就是让人高兴的，何必搞得那么伤感呢？现在有《仙剑奇侠传》的电视剧了，看了，就不用在玩了。

其它的游戏玩的少，现在玩的比较多的就是《反恐精英》和《流行蝴蝶剑》了，很简单的两点：现在时间是越来越少了，不希望花费太多时间玩游戏；游戏就是要人快乐，这两个游戏我玩得还不错，杀电脑是小菜，心情不好的时候，杀杀人，现现自己和枪法和武艺，也挺快乐的。

关于网络游戏，我感触很深，因为我差一点就曾经陷入网络游戏的漩涡中。那是关键的初四上学期，同学们都在玩《传奇》，几个好朋友也邀请我玩，我试着去玩了几次，觉得很不错，就像在一个虚拟的世界中，我们靠着自己的努力升级，成为强者。后来发展到，每个中午我都要在网吧中待上一个小时，有时甚至星期天玩一上午。可是，我没有象有些同学逃课，上课依然听讲，可以有点力不从心，那段时间我都成绩也落了下来，知道有一次这事情被老师家访了，我之后就再也没有去过网吧了。随后成绩也就上来了。回想《传奇》中的世间，我们在打怪练级，完成任务获得奖励，我们希望成为强制，难道人生也不是这样吗，我们在学习，就像游戏中学习的技能一样，我们战胜心魔不就像里面的打怪吗，每次打怪之后，我们的经验值会上升，升到一些时候就升级，我们打败心魔之后不也是这样吗，心里就有新的感觉、新的体会，我们会对人生有新的感悟，这不就是无形的经验吗？只是游戏中的经验可以看到，我们可以了解何时升级，何时换装备。而生活中的经验是无形的，我们的武器的装备也是不变的，它就是我们的思维。还有一个最大的区别，在游戏中一旦你被打败，你可以很简单的重生，但是一旦你在生活中被打败，也许你就只能永远沉沦在你的心魔中了。
　

网络

我们家现在已经不能上网了，因为连电话都被拆了，这也是我的一个遗憾之一，没有了网络，也就没有了丰富的知识来源，没有了便捷的机会和别人交流了。

以前我们家中是有电话的，记得初二的时候，父亲带回来一个 Modem，那是所用的是 663 拨号上网，很贵，我每次都是很小心地很节约的使用，现在仍然保留着这个习惯，无论是在网吧还是别人家上网，如果连上了 Internet 我就必须使用，否则就断开，那是使用网也就只是下下软件，看看新闻而已。现在不同了，有了 BT，想怎么下就怎么下，什么东西都有。

关于 BT，我算是用得比较早的人了，在初中毕业的时候，我就在表哥家上网，他家的网很快，是 10MB 的，可以他们都不会利用，而且他们家也是最先拥有刻录机的，所以每次下的东西都可以刻盘拿回家。那是我便开始在他们家中下东西了，最开始使用的是 HTTP 方式下载，可是有些软件和游戏使用 HTTP 方式下载很慢，最后我便找到了新兴的 BT 下载方式，的确很快，当时还没有 BitComet，所以下的东西不是很多。后来有了 BitComet，速度有的大幅增加（PS：他们家是内网），下的东西也就多了。知道现在，我的大部分资源都是在他家下的。真希望何时我们家也能装上宽带，那就不用每次跑到别家去了。

QQ 我很少用，应为几乎讨论的问题都是在论坛上讨论或者通过 Email 讨论，QQ 只是一个和同学联系的工具而已。现在没有电话了，网也几乎很少上了。
　

未来

没办法，要高考了，也许这是我高中生涯的最后一个软件，但是它决不是我的最后一个软件，高考决定命运，我现在也要开始冲一把了，为了未来，也为了高考，也是时间开始扎扎实实地学习了，虽然编程提高了我的英语水平、提高了我的知识面、培养了我的自学能力。但是现在也该是舍的时候，何去何从，孰重孰轻，现在都是要将100% 的精力花在复习上了。

至于下一版本，高考完了在说吧。
